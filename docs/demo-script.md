# Demo Script

This is the 3-minute demo flow for Authrix.

The goal is to clearly show:
- real data to insight to action
- secure, permissioned AI behavior
- a clean, believable system

Clarity matters more than complexity.

---

## 0:00 to 0:20 — Hook

Say:

“Managing engineering work across tools is messy. Updates are scattered, tasks get lost, and letting AI act on real systems can be risky.”

“Authrix solves that by using secure, permissioned AI agents that turn activity into action while keeping the user in control.”

---

## 0:20 to 0:40 — Login and setup

Show:
- Auth0 login or authenticated state
- GitHub connection status

Say:

“Authrix connects to real systems, starting with GitHub, and uses strict permission boundaries between agents.”

---

## 0:40 to 1:00 — Trigger the workflow

User action:
- open dashboard
- or click refresh

Say:

“Let’s look at what happened this week.”

---

## 1:00 to 1:30 — Weekly Summary

Show:
- Weekly Summary card

Say:

“The Engineer Agent looks at engineering activity and generates a structured summary of what changed.”

Optional emphasis:
- PRs merged
- features worked on
- fixes shipped
- current momentum

---

## 1:30 to 1:55 — Suggested Tasks

Show:
- Suggested Tasks card

Say:

“The Task Agent takes only the summary output, not the full raw data, and turns it into actionable next steps.”

This is a good place to emphasize minimal data sharing between agents.

---

## 1:55 to 2:15 — Cost and Risk

Show:
- API Spend / Risk card

Say:

“The DevOps Agent monitors usage and flags possible cost or risk concerns, so teams can catch problems early.”

---

## 2:15 to 2:40 — Approval moment

User clicks:
- Create tasks
- Approve action
- or similar CTA

Approval modal appears.

Say:

“Authrix does not let agents act blindly. Any write action is approval-gated, so users stay in control.”

This is one of the most important moments in the demo.

---

## 2:40 to 2:55 — Controlled execution

User approves action.

Show:
- task creation confirmation
- approval queue update
- success state

Say:

“Agents can help plan and act, but execution still happens through a controlled layer with explicit approval.”

---

## 2:55 to 3:00 — Close

Say:

“Authrix makes multi-agent systems usable in real workflows by combining automation with security, transparency, and control.”

---

## Key message

Authrix is not just a system of agents.

It is a system that makes AI agents safe to use on real tools and real workflows.

---

## Notes for presenters

- Keep the pacing steady
- Do not overexplain the architecture
- Emphasize the approval moment
- Emphasize scoped access and minimal data sharing
- Show, don’t tell
- If something is mocked, present it confidently and clearly
